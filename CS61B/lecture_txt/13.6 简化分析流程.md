# 13.6 简化分析流程
事情并不简单

**传统分析流程总结（耗时费力）**
- 构建所有可能操作的精确计数表（需要大量精力！）
- 使用四项简化原则将表格转换为最坏情况增长阶数

现在我们将提出一种完全避免构建计数表的替代方法！

简化分析流程包含：
1. 选择成本模型（即需要统计的代表性操作）
2. 通过以下方式确定代表操作的增长阶数：
   - 精确统计后去除无关项，或
   - 通过实践积累的直觉/观察直接判断增长阶数

#### 示例：嵌套for循环分析 - 精确计数法
求`dup1`最坏情况运行时增长阶数

```java
int N = A.length;
for (int i = 0; i < N; i += 1)
   for (int j = i + 1; j < N; j += 1)
      if (A[i] == A[j])
         return true;
return false;
```


选择"=="操作次数作为成本模型。观察循环结构：
- 外层i=0时，内层循环执行j=N-1次
- i=1时，内层循环执行j=N-2次
- 依此类推，总操作次数为：

$cost = 1 + 2 + 3 + \dots + (N-2) + (N-1)$

简化为$N(N-1)/2$（[推导过程](https://en.wikipedia.org/wiki/1_%2B_2_%2B_3_%2B_4_%2B_%E2%8B%AF)）。去除低阶项和常数后得到最坏情况增长阶数$N^2$。
