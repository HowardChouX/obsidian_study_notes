# 类定义

>可以把「类」想象成 **产品设计图纸**，而根据图纸生产出来的手机就是「对象」。
>用图纸对成员数据和成员函数进行操作；


```c++
//  类比设计图纸（类）
class 手机图纸 {
private:    // 内部机密（封装）
    主板电路设计;
    电池配方;

public:     // 用户能用的功能
    string 型号;
    string 颜色;
    
    void 打电话() {
        启动通信模块();
    }
    
    void 拍照() {
        调用摄像头();
    }
};

// 2. 生产具体手机（创建对象）
手机图纸 我的手机;         // 造出一台裸机
我的手机.型号 = "Mate60";  // 贴上型号标签
我的手机.打电话();         // 使用功能

// 3. 不能直接碰的部件（封装性）
我的手机.电池配方 = "xxx";  // ❌ 禁止操作（private成员）
```

```c++
[[include]] <iostream>

class Player // 相当于「赛车遥控器」
{
private:// 隐藏在遥控器内部的电路（用户不能直接碰）
    int x = 0, y = 0;  // 从坐标原点出发
    int speed = 2;      // 默认速度档位2档

public:// 遥控器上的按钮（用户可操作部分）
    void Move(int xa, int ya) // 方向键
    {
        x += xa * speed; 按方向 × 速度档位 = 实际移动距离
        y += ya * speed;
        std::cout << "移动到 (" << x << "," << y << ")\n";
    }
};

int main() 
{
    Player player;
    player.Move(1, -1);  // 向右上移动（实际坐标：2,-2）
    player.Move(0, 1);   // 向上移动（实际坐标：2,0）
    std::cin.get();
}```

# 类和结构体对比
> 数据处理倾向于类，功能集合倾向于结构体，继承一般用类不用结构体。

| 特性         | `class`             | `struct`    |
| ---------- | ------------------- | ----------- |
| **默认访问权限** | `private`           | `public`    |
| **继承默认权限** | `private` 继承        | `public` 继承 |
| **典型用途**   | 封装复杂对象              | 数据聚合体       |
| **类型语义**   | 主动对象（带==行为==）       | 被动==数据==集合  |
| **类型标识**   | 强调类型不变式（Invariants） | 强调数据成员自由组合  |
