
# 存/取规则（后进先出)(LIFO)

- **存**：新羽毛球只能从**筒口塞进去**（最后放进去的球在**最上面**）。
- **取**：每次只能从**筒口拿最上面**的球（不能直接掏中间的球）。
- **简称**：**LIFO**（Last In, First Out，后进先出）。

---

#  生活中的例子

## 场景1：叠盘子

- **洗碗时**：每洗好一个盘子就叠到最上面（`push`压入操作）。
- **用餐时**：总是拿最顶上的干净盘子（`pop`弹出操作）。
- **偷偷看**：瞄一眼最上面的盘子花纹（`peek`操作）。

## 场景2：浏览器返回键

- 你依次访问：**主页 → 新闻 → 体育 → 比分**
- 按返回键时，顺序是：**比分 ← 体育 ← 新闻 ← 主页**（反向退出）。

#  计算机中的核心用途

## 函数调用栈（打电话场景）

假设主函数`main`调用`函数A`，`函数A`又调用`函数B`：  
1️⃣ `main`拨通`A` → 把`main`的现场信息（如执行位置）**压入栈**，暂停。  
2️⃣ `A`拨通`B` → 把`A`的现场信息**压入栈**，暂停。  
3️⃣ `B`执行完 → **弹出栈顶**，回到`A`继续执行。  
4️⃣ `A`执行完 → **弹出栈顶**，回到`main`继续。

# 内存管理（栈变量）

- 局部变量（如函数内的`int x`）存在栈里，函数结束自动释放（像自动退房）。
- 快速高效，但容量小（对比堆内存像「仓库」，需要手动管理）。

---

#  栈的优缺点

|优点|缺点|
|---|---|
|操作极快（只动栈顶）|容量有限（易溢出）|
|自动管理内存（无泄漏）|不能随机访问中间数据|
